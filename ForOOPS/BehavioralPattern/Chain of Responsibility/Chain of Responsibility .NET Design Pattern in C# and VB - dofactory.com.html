Chain of Responsibility


 Definition
 UML diagram
 Participants
 Structural code in C#
 Real-world code in C#
 .NET Optimized code in C#


Definition

Avoid coupling the sender of a request to its receiver by giving more than one object a chance to handle the request. Chain the receiving objects and pass the request along the chain until an object handles it.

Frequency of use:
Medium low



UML class diagram





Participants


    The classes and objects participating in this pattern are:

  • Handler   (Approver)
    • defines an interface for handling the requests
    • (optional) implements the successor link
  • ConcreteHandler   (Director, VicePresident, President)
    • handles requests it is responsible for
    • can access its successor
    • if the ConcreteHandler can handle the request, it does so; otherwise it forwards the request to its successor
  • Client   (ChainApp)
    • initiates the request to a ConcreteHandler object on the chain


Structural code in C#


This structural code demonstrates the Chain of Responsibility pattern in which several linked objects (the Chain) are offered the opportunity to respond to a request or hand it off to the object next in line. 

  1.                  
  3. using System;
  4.  
  5. namespace DoFactory.GangOfFour.Chain.Structural
  6. {
  7.   /// <summary>
  8.   /// MainApp startup class for Structural
  9.   /// Chain of Responsibility Design Pattern.
  10.   /// </summary>
  11.   class MainApp
  12.   {
  13.     /// <summary>
  14.     /// Entry point into console application.
  15.     /// </summary>
  16.     static void Main()
  17.     {
  18.       // Setup Chain of Responsibility
  19.       Handler h1 = new ConcreteHandler1();
  20.       Handler h2 = new ConcreteHandler2();
  21.       Handler h3 = new ConcreteHandler3();
  22.       h1.SetSuccessor(h2);
  23.       h2.SetSuccessor(h3);
  24.  
  25.       // Generate and process request
  26.       int[] requests = { 2, 5, 14, 22, 18, 3, 27, 20 };
  27.  
  28.       foreach (int request in requests)
  29.       {
  30.         h1.HandleRequest(request);
  31.       }
  32.  
  33.       // Wait for user
  34.       Console.ReadKey();
  35.     }
  36.   }
  37.  
  38.   /// <summary>
  39.   /// The 'Handler' abstract class
  40.   /// </summary>
  41.   abstract class Handler
  42.   {
  43.     protected Handler successor;
  44.  
  45.     public void SetSuccessor(Handler successor)
  46.     {
  47.       this.successor = successor;
  48.     }
  49.  
  50.     public abstract void HandleRequest(int request);
  51.   }
  52.  
  53.   /// <summary>
  54.   /// The 'ConcreteHandler1' class
  55.   /// </summary>
  56.   class ConcreteHandler1 : Handler
  57.   {
  58.     public override void HandleRequest(int request)
  59.     {
  60.       if (request >= 0 && request < 10)
  61.       {
  62.         Console.WriteLine("{0} handled request {1}",
  63.           this.GetType().Name, request);
  64.       }
  65.       else if (successor != null)
  66.       {
  67.         successor.HandleRequest(request);
  68.       }
  69.     }
  70.   }
  71.  
  72.   /// <summary>
  73.   /// The 'ConcreteHandler2' class
  74.   /// </summary>
  75.   class ConcreteHandler2 : Handler
  76.   {
  77.     public override void HandleRequest(int request)
  78.     {
  79.       if (request >= 10 && request < 20)
  80.       {
  81.         Console.WriteLine("{0} handled request {1}",
  82.           this.GetType().Name, request);
  83.       }
  84.       else if (successor != null)
  85.       {
  86.         successor.HandleRequest(request);
  87.       }
  88.     }
  89.   }
  90.  
  91.   /// <summary>
  92.   /// The 'ConcreteHandler3' class
  93.   /// </summary>
  94.   class ConcreteHandler3 : Handler
  95.   {
  96.     public override void HandleRequest(int request)
  97.     {
  98.       if (request >= 20 && request < 30)
  99.       {
  100.         Console.WriteLine("{0} handled request {1}",
  101.           this.GetType().Name, request);
  102.       }
  103.       else if (successor != null)
  104.       {
  105.         successor.HandleRequest(request);
  106.       }
  107.     }
  108.   }
  109. }

Real-world code in C#


This real-world code demonstrates the Chain of Responsibility pattern in which several linked managers and executives can respond to a purchase request or hand it off to a superior. Each position has can have its own set of rules which orders they can approve. 

  1.                  
  3. using System;
  4.  
  5. namespace DoFactory.GangOfFour.Chain.RealWorld
  6. {
  7.   /// <summary>
  8.   /// MainApp startup class for Real-World 
  9.   /// Chain of Responsibility Design Pattern.
  10.   /// </summary>
  11.   class MainApp
  12.   {
  13.     /// <summary>
  14.     /// Entry point into console application.
  15.     /// </summary>
  16.     static void Main()
  17.     {
  18.       // Setup Chain of Responsibility
  19.       Approver larry = new Director();
  20.       Approver sam = new VicePresident();
  21.       Approver tammy = new President();
  22.  
  23.       larry.SetSuccessor(sam);
  24.       sam.SetSuccessor(tammy);
  25.  
  26.       // Generate and process purchase requests
  27.       Purchase p = new Purchase(2034, 350.00, "Assets");
  28.       larry.ProcessRequest(p);
  29.  
  30.       p = new Purchase(2035, 32590.10, "Project X");
  31.       larry.ProcessRequest(p);
  32.  
  33.       p = new Purchase(2036, 122100.00, "Project Y");
  34.       larry.ProcessRequest(p);
  35.  
  36.       // Wait for user
  37.       Console.ReadKey();
  38.     }
  39.   }
  40.  
  41.   /// <summary>
  42.   /// The 'Handler' abstract class
  43.   /// </summary>
  44.   abstract class Approver
  45.   {
  46.     protected Approver successor;
  47.  
  48.     public void SetSuccessor(Approver successor)
  49.     {
  50.       this.successor = successor;
  51.     }
  52.  
  53.     public abstract void ProcessRequest(Purchase purchase);
  54.   }
  55.  
  56.   /// <summary>
  57.   /// The 'ConcreteHandler' class
  58.   /// </summary>
  59.   class Director : Approver
  60.   {
  61.     public override void ProcessRequest(Purchase purchase)
  62.     {
  63.       if (purchase.Amount < 10000.0)
  64.       {
  65.         Console.WriteLine("{0} approved request# {1}",
  66.           this.GetType().Name, purchase.Number);
  67.       }
  68.       else if (successor != null)
  69.       {
  70.         successor.ProcessRequest(purchase);
  71.       }
  72.     }
  73.   }
  74.  
  75.   /// <summary>
  76.   /// The 'ConcreteHandler' class
  77.   /// </summary>
  78.   class VicePresident : Approver
  79.   {
  80.     public override void ProcessRequest(Purchase purchase)
  81.     {
  82.       if (purchase.Amount < 25000.0)
  83.       {
  84.         Console.WriteLine("{0} approved request# {1}",
  85.           this.GetType().Name, purchase.Number);
  86.       }
  87.       else if (successor != null)
  88.       {
  89.         successor.ProcessRequest(purchase);
  90.       }
  91.     }
  92.   }
  93.  
  94.   /// <summary>
  95.   /// The 'ConcreteHandler' class
  96.   /// </summary>
  97.   class President : Approver
  98.   {
  99.     public override void ProcessRequest(Purchase purchase)
  100.     {
  101.       if (purchase.Amount < 100000.0)
  102.       {
  103.         Console.WriteLine("{0} approved request# {1}",
  104.           this.GetType().Name, purchase.Number);
  105.       }
  106.       else
  107.       {
  108.         Console.WriteLine(
  109.           "Request# {0} requires an executive meeting!",
  110.           purchase.Number);
  111.       }
  112.     }
  113.   }
  114.  
  115.   /// <summary>
  116.   /// Class holding request details
  117.   /// </summary>
  118.   class Purchase
  119.   {
  120.     private int _number;
  121.     private double _amount;
  122.     private string _purpose;
  123.  
  124.     // Constructor
  125.     public Purchase(int number, double amount, string purpose)
  126.     {
  127.       this._number = number;
  128.       this._amount = amount;
  129.       this._purpose = purpose;
  130.     }
  131.  
  132.     // Gets or sets purchase number
  133.     public int Number
  134.     {
  135.       get { return _number; }
  136.       set { _number = value; }
  137.     }
  138.  
  139.     // Gets or sets purchase amount
  140.     public double Amount
  141.     {
  142.       get { return _amount; }
  143.       set { _amount = value; }
  144.     }
  145.  
  146.     // Gets or sets purchase purpose
  147.     public string Purpose
  148.     {
  149.       get { return _purpose; }
  150.       set { _purpose = value; }
  151.     }
  152.   }
  153. }
  154.  
  155.
